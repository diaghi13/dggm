## üìä Flusso Dettagliato: CREATE Warehouse

```
1. USER ACTION
   Frontend ‚Üí POST /api/v1/warehouses
   Body: { code: "WH001", name: "Main Warehouse", ... }

2. LARAVEL ROUTING
   routes/api.php ‚Üí WarehouseController@store

3. CONTROLLER (HTTP Layer)
   ‚îú‚îÄ Authorization: $this->authorize('create', Warehouse::class)
   ‚îú‚îÄ Validation: StoreWarehouseRequest (FormRequest)
   ‚îî‚îÄ Delegate: $this->service->create($request->validated())

4. SERVICE (Orchestrator)
   ‚îú‚îÄ Convert to DTO: WarehouseData::fromRequest($data)
   ‚îî‚îÄ Delegate: $this->createAction->execute($dto)

5. ACTION (Business Logic)
   ‚îú‚îÄ START DB TRANSACTION
   ‚îú‚îÄ Repository::create($dto) ‚Üí INSERT INTO warehouses
   ‚îú‚îÄ DISPATCH: WarehouseCreated::dispatch($warehouse, metadata)
   ‚îî‚îÄ COMMIT DB TRANSACTION

6. EVENT SYSTEM (Automatic)

   ‚îú‚îÄ Listener 1: LogWarehouseActivity::handleCreated()
   ‚îÇ   ‚îî‚îÄ Log::info("Warehouse created", ...)
   ‚îú‚îÄ Listener 2: UpdateWarehouseCache::handleCreated()
   ‚îÇ   ‚îî‚îÄ Cache::forget('warehouses:all')
   ‚îî‚îÄ Broadcasting: Pusher/Reverb WebSocket
       ‚îî‚îÄ Send to channel 'warehouses'

   ‚îî‚îÄ (No queued listeners for Created event)

     });

     "data": { id: 1, code: "WH001", ... },

   }

## üî• Flusso Dettagliato: LOW STOCK Alert

```
1. INVENTORY UPDATE
   Stock movement ‚Üí Inventory::quantity_available = 5
   Minimum stock = 10

2. BUSINESS LOGIC CHECK (in WarehouseService or StockMovementService)
   if ($inventory->quantity_available <= $inventory->minimum_stock) {
       InventoryLowStock::dispatch($inventory, $warehouse);
   }

3. EVENT DISPATCHED
   InventoryLowStock::dispatch($inventory, $warehouse)

4. EVENT SYSTEM
   ‚îú‚îÄ Listener: SendLowStockAlert (ShouldQueue - Async!)
   ‚îÇ   ‚îú‚îÄ Added to QUEUE (Redis/Database)
   ‚îÇ   ‚îî‚îÄ Will process in background
   ‚îî‚îÄ Broadcasting: WebSocket to frontend
       ‚îî‚îÄ Channel: 'warehouses' and 'warehouse.{id}'

   ‚îú‚îÄ Get warehouse manager: $event->warehouse->manager
   ‚îú‚îÄ Send email: Mail::to($manager)->send(LowStockAlert)
   ‚îú‚îÄ Send notification: Notification::send(...)

## üéØ Benefici di questa Architettura

### 1. Separazione delle Responsabilit√†
```
Controller  ‚Üí HTTP concerns (auth, validation, response)
Service     ‚Üí Orchestrazione (quale Action/Query chiamare)
Action      ‚Üí Business logic + DB transaction + Events
Query       ‚Üí Read operations complesse
Repository  ‚Üí Data access abstraction
Events      ‚Üí Side-effects decoupling
Listeners   ‚Üí Audit, cache, notifications (async)
```

### 2. Testabilit√† Isolata
```php
// Test Action without HTTP
$action = app(CreateWarehouseAction::class);
$warehouse = $action->execute($dto);
expect($warehouse)->toBeInstanceOf(Warehouse::class);

// Test Events
Event::fake([WarehouseCreated::class]);
$action->execute($dto);
Event::assertDispatched(WarehouseCreated::class);

// Test Listener
$listener = new SendLowStockAlert();
$listener->handle($event);
Mail::assertSent(LowStockAlert::class);
```

### 3. Scalabilit√†
- **Horizontal**: Aggiungi listener senza modificare Actions
- **Vertical**: Cambia DB (Mongo, Postgres) senza toccare Service/Actions
- **Multi-tenant**: Repository diversi per tenant diversi

### 4. Performance
- **Cache Layer**: UpdateWarehouseCache invalida solo ci√≤ che serve
- **Async Processing**: SendLowStockAlert non blocca la request
- **Broadcasting**: Frontend aggiornato in real-time senza polling

### 5. Manutenibilit√†
- Modifiche locali ‚Üí nessun side-effect
- Nuovo listener ‚Üí aggiungi in EventServiceProvider
- Nuovo database ‚Üí implementa WarehouseXRepository
- Nuova feature ‚Üí aggiungi Action/Query

## üöÄ Esempio: Aggiungere un Nuovo Listener

**Requirement**: Inviare webhook a sistema esterno quando warehouse creato.

```php
// 1. Crea Listener
php artisan make:listener SendWebhookToExternalSystem

// 2. Implementa
class SendWebhookToExternalSystem implements ShouldQueue
{
    public function handle(WarehouseCreated $event): void
    {
        Http::post('https://external-system.com/webhooks', [
            'event' => 'warehouse.created',
            'warehouse_id' => $event->warehouse->id,
            'warehouse_code' => $event->warehouse->code,
        ]);
    }
}

// 3. Registra in EventServiceProvider
protected $listen = [
    WarehouseCreated::class => [
        UpdateWarehouseCache::class,
        SendWebhookToExternalSystem::class, // ‚Üê Aggiunto!
    ],
];
```

**FATTO!** Nessuna modifica a Controller, Service, Action. Zero side-effects.
